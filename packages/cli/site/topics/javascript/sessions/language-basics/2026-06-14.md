# Language Basics - Session Notes (2026-06-14)

## Variables and Data Types

Today I covered the fundamental building blocks of JavaScript data:

- **`var` vs `let` vs `const`**: `let` and `const` are block-scoped and should be preferred over `var`. `const` prevents reassignment but not mutation of objects/arrays.
- **Primitive types**: `string`, `number`, `bigint`, `boolean`, `undefined`, `symbol`, `null`
- **Reference types**: Objects, arrays, functions — stored by reference, compared by reference
- **Template strings**: Use backticks and `${}` for interpolation, support multi-line strings
- **Symbol**: Guaranteed unique, useful for object property keys to avoid collisions
- **BigInt**: For integers larger than `Number.MAX_SAFE_INTEGER` (2^53 - 1)

## Key Takeaways

- Use `const` by default, `let` when reassignment is needed, never `var`
- `typeof null === 'object'` is a historical bug
- Template literals are more readable than `+` concatenation
