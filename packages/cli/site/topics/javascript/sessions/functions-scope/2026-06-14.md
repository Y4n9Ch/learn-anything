# Functions and Scope - Session Notes (2026-06-14)

## Function Definitions

Explored the different ways to define functions in JavaScript:

### Function Declaration

```js
function greet(name) {
  return `Hello, ${name}!`;
}
```

- Hoisted to top of scope — can be called before declaration
- Named, which helps with stack traces

### Function Expression

```js
const greet = function (name) {
  return `Hello, ${name}!`;
};
```

- NOT hoisted — cannot be called before definition
- Can be anonymous or named

### Arrow Functions

```js
const greet = (name) => `Hello, ${name}!`;
```

- No own `this`, `arguments`, `super`, or `new.target`
- Lexical `this` binding — inherits from enclosing scope
- Implicit return when using concise body (no braces)

### Parameters

- **Default values**: `function f(x = 1) {}`
- **Rest parameters**: `function f(...args) {}` — collects remaining args into array
- The `arguments` object is not available in arrow functions

## Key Takeaways

- Use arrow functions for callbacks and short functions
- Use function declarations for top-level named functions
- Rest parameters are preferred over the `arguments` object
