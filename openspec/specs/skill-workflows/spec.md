# Skill Workflows Specification

5 learning workflows (topic/explain/practice/review/status) use state.json as the single data source.

## MODIFIED Requirements

### Requirement: /learn New Topic Creation

The system SHALL initialize a new learning topic by writing `state.json` (containing the full domains/concepts hierarchical structure), replacing the previous dual-file creation of knowledge-map.md + state.yaml.

Required initial field values for concepts:

- `status: "unexplored"`
- `confidence: 0`
- `practice_count: 0`
- `explain_count: 0`
- `last_explained: null`
- `last_practiced: null`
- `details: []` (or populated with third-level sub-concept names based on the knowledge map structure)

#### Scenario: Create new topic

- **WHEN** the user enters `/learn rust`
- **AND** the `.learn/topics/rust/` directory does not exist
- **THEN** AI creates `state.json` containing Rust's domains/concepts hierarchical structure
- **AND** AI runs `render.mjs` to generate `knowledge-map.md`
- **AND** AI presents the knowledge map to the user

#### Scenario: No standalone knowledge-map.md instructions

- **WHEN** AI follows skill instructions to create a new topic
- **THEN** skill instructions do not include a step to manually write knowledge-map.md (that file is generated by render.mjs)

### Requirement: /learn Load Existing Topic

The system SHALL only read `state.json` to understand an existing topic's structure and progress, no longer reading both knowledge-map.md and state.yaml simultaneously.

#### Scenario: Open existing topic

- **WHEN** the user enters `/learn javascript`
- **AND** `.learn/topics/javascript/state.json` already exists
- **THEN** AI only reads `state.json` (does not read knowledge-map.md)
- **AND** AI calculates mastery statistics from state.json's domains/concepts structure
- **AND** AI provides personalized recommendations based on state data

### Requirement: /learn-explain Concept Explanation

The system SHALL only fetch concept structure from `state.json`, update concept status fields (`status`, `last_explained`, `explain_count`, `confidence`), and run `render.mjs` after updates to regenerate knowledge-map.md.

**BREAKING**: Remove references to the `last_session` field; use `last_explained` consistently.

#### Scenario: Explain new concept

- **WHEN** the user enters `/learn-explain closures`
- **AND** the concept has `status: "unexplored"` in state.json
- **THEN** after AI explains, it updates state.json:
  - `status` → `"in_progress"`
  - `last_explained` → current date
  - `explain_count` + 1
  - `confidence` +0.05~0.1 (if understanding is good)
- **AND** AI runs `render.mjs` to update knowledge-map.md

#### Scenario: No longer reads knowledge-map.md

- **WHEN** AI executes the /learn-explain workflow
- **THEN** AI only reads state.json to locate concept position and current status

### Requirement: /learn-practice Concept Practice

The system SHALL only fetch concept status and assessed difficulty from `state.json`, update concept status fields (`status`, `last_practiced`, `practice_count`, `confidence`), and run `render.mjs` after updates.

#### Scenario: Post-practice assessment

- **WHEN** the user completes practice and receives an excellent rating
- **AND** the concept's `confidence` rises to > 0.7 and `practice_count` ≥ 2
- **THEN** AI updates state.json: `status` → `"mastered"`, `confidence` +0.1~0.15, `practice_count` +1, `last_practiced` → current date
- **AND** AI runs `render.mjs`

#### Scenario: Post-practice needs reinforcement

- **WHEN** the user's practice is mostly correct but has edge case issues
- **THEN** AI updates state.json: `status` → `"needs_practice"`, `confidence` +0.05, `practice_count` +1
- **AND** AI runs `render.mjs`

### Requirement: /learn-review Review Analysis

The system SHALL only read `state.json`, calculating mastery heatmap and spaced repetition priorities from the domains/concepts hierarchical structure.

#### Scenario: Generate review report

- **WHEN** the user enters `/learn-review javascript`
- **THEN** AI reads state.json
- **AND** calculates statistics from domains and concepts (mastered/in_progress/needs_practice/unexplored counts)
- **AND** calculates priority using the formula `(1 - confidence) * (days_since_last_practice + 1) * w`

### Requirement: /learn-status Status Display

The system SHALL only read `state.json`, displaying the knowledge map heatmap and overview panel. Review and status are read-only operations that do not execute `render.mjs`.

#### Scenario: Display learning status

- **WHEN** the user enters `/learn-status javascript`
- **THEN** AI calculates statistics from state.json and displays them
- **AND** display includes mastered/active/practice/unexplored counts and progress percentage
